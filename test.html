Database Management System (DBMS)

Introduction

A Database Management System (DBMS) is software designed to store, retrieve, define, and manage data in a database. It serves as an interface between end-users and the database, ensuring data is consistently organized and remains accessible.

Objectives

  • Data integrity and security
  • Efficient data storage and retrieval
  • Minimized data redundancy
  • Concurrency control
  • Backup and recovery mechanisms

Advantages

  • Improved data sharing
  • Enhanced data security
  • Reduced data inconsistency
  • Better decision making
  • Increased productivity

Disadvantages

  • Complexity in setup and management
  • High cost of implementation
  • Performance overhead
  • Vulnerability to failures
  • Requires specialized personnel

DBMS Fundamentals

1. What is Database and DBMS? List out the advantages and disadvantages of DBMS.

Database & DBMS

Database: An organized collection of structured data stored electronically for efficient access, management, and updates.

DBMS (Database Management System): Software that manages databases (e.g., MySQL, Oracle). It provides tools to define, create, query, update, and administer databases.

Advantages

  • Reduced Data Redundancy: Minimizes duplicate data through normalization and centralized control
  • Improved Data Integrity: Enforces constraints (e.g., unique keys) to ensure accuracy
  • Enhanced Security: Role-based access control, encryption, and audit trails
  • Data Sharing and Concurrency: Supports simultaneous access by multiple users
  • Backup and Recovery: Automated tools for data restoration after failures
  • Data Independence: Applications are insulated from changes in storage structures

Disadvantages

  • High Setup Cost: Expensive licensing, hardware, and specialized personnel
  • Complexity: Requires expertise in design, implementation, and maintenance
  • Performance Overhead: Resource-intensive operations for large datasets
  • Scalability Limitations: Vertical scaling can be challenging in centralized systems
  • Single Point of Failure: Centralized databases risk complete system downtime

3. Explain the different models of DBMS with advantages and disadvantages.

Database Models

1. Hierarchical Model

  • Structure: Tree-like (parent-child relationships)
  • Advantages: Fast access for hierarchical data, efficient for 1:N relationships
  • Disadvantages: Inflexible; cannot model many-to-many relationships, complex to modify

2. Network Model

  • Structure: Graph-like (multiple parent-child connections)
  • Advantages: Handles complex relationships, better than hierarchical for M:N relationships
  • Disadvantages: High complexity in design and queries, difficult to modify

3. Relational Model

  • Structure: Tables (rows and columns)
  • Advantages: Simple design, SQL support, flexible queries, data independence
  • Disadvantages: Performance issues with complex joins, requires careful design

4. Object-Oriented Model

  • Structure: Objects (attributes and methods)
  • Advantages: Suitable for complex data, supports inheritance and encapsulation
  • Disadvantages: Steep learning curve, limited industry adoption, performance issues
4. What is a relational database? How is it different from other database models?

Relational Database

A relational database stores data in tables (relations) with rows (tuples) and columns (attributes). Relationships between tables are established using keys (primary and foreign keys).

Differences from Other Models

  • vs. Hierarchical/Network Models: Uses logical links (keys) instead of physical pointers
  • vs. Object-Oriented Models: Avoids complex inheritance structures; relies on tables
  • vs. NoSQL: Requires a fixed schema; strict ACID compliance
  • vs. Document Databases: Uses structured tables rather than JSON-like documents
5. What is data redundancy? How does DBMS help in reducing it?

Data Redundancy

Data Redundancy refers to unnecessary duplication of data in a database (e.g., storing the same customer address in multiple tables).

How DBMS Reduces Redundancy

  • Normalization: Splits tables to store data once (e.g., separate Customers and Orders tables)
  • Foreign Keys: References unique IDs instead of repeating data (e.g., CustomerID in Orders)
  • Centralized Storage: Single source of truth for critical data
  • Data Integrity Constraints: Ensures consistency across related tables

7. Who is Database Administrator (DBA)? What are the major responsibilities of DBA?

Database Administrator (DBA)

A Database Administrator (DBA) is a specialist responsible for managing and maintaining database systems to ensure their performance, integrity, and security.

Major Responsibilities

  • Database Design: Planning and implementing database structures
  • Security Management: Implementing access controls, encryption, and auditing
  • Performance Tuning: Optimizing queries, indexing, and resource allocation
  • Backup and Recovery: Creating and testing disaster recovery plans
  • Data Integrity: Ensuring accuracy and consistency through constraints
  • Schema Changes: Managing database modifications using DDL
  • Capacity Planning: Forecasting storage and performance requirements
8. Define normalization. Explain 1NF, 2NF and 3NF with suitable examples.

Normalization

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity by dividing large tables into smaller, related tables.

1NF (First Normal Form)

Rule: Atomic values; no repeating groups

Example: Convert Students (ID, Name, [Subject1, Subject2]) to Students (ID, Name, Subject)

2NF (Second Normal Form)

Rule: 1NF + no partial dependencies (all non-key attributes depend on the entire primary key)

Example: Split Orders (OrderID, ProductID, ProductName, Price) into Orders (OrderID, ProductID) and Products (ProductID, ProductName, Price)

3NF (Third Normal Form)

Rule: 2NF + no transitive dependencies (non-key attributes depend only on the primary key)

Example: Split Employees (EmpID, Name, Dept, DeptLocation) into Employees (EmpID, Name, Dept) and Departments (Dept, DeptLocation)

9. Explain the terms: primary key, foreign key, and candidate key with examples.

Keys in DBMS

Primary Key: Unique identifier for a table (e.g., StudentID in Students table)

  • Must be unique and not null
  • Example: ISBN in Books table

Foreign Key: Attribute linking to another table's primary key (e.g., DeptID in Employees table references Departments table)

  • Maintains referential integrity
  • Example: CustomerID in Orders table

Candidate Key: Column(s) eligible to be a primary key (e.g., Email or SSN in Users table)

  • Must be unique and not null
  • A table can have multiple candidate keys
10. What is SQL? Explain its components and common functions.

SQL (Structured Query Language)

SQL is the standard language for managing relational databases, used for querying, updating, and managing data.

SQL Components

  • DDL (Data Definition Language): CREATE, ALTER, DROP (define structure)
  • DML (Data Manipulation Language): SELECT, INSERT, UPDATE, DELETE (manage data)
  • DCL (Data Control Language): GRANT, REVOKE (access control)
  • TCL (Transaction Control Language): COMMIT, ROLLBACK (transaction management)

Common SQL Functions

  • Aggregate: COUNT(), SUM(), AVG(), MAX(), MIN()
  • String: CONCAT(), SUBSTRING(), UPPER(), LOWER()
  • Date: NOW(), DATE_ADD(), DATEDIFF()
  • Mathematical: ROUND(), ABS(), RAND()
11. Write SQL commands to perform the following operations:
-- a. Create a table named students with fields: Id, Name, Class, and Marks
CREATE TABLE students (
  Id INT PRIMARY KEY,
  Name VARCHAR(50) NOT NULL,
  Class CHAR(3),
  Marks INT CHECK (Marks >= 0)
);

-- b. Insert records into the students table
INSERT INTO students (Id, Name, Class, Marks)
VALUES
  (1, 'Raj', '10A', 92),
  (2, 'Anita', '9B', 78),
  (3, 'Sam', '11C', 85);

-- c. Display all records from the students table
SELECT * FROM students;

-- d. Update the marks of a student named 'Anita' to 85
UPDATE students
SET Marks = 85
WHERE Name = 'Anita';

-- e. Delete the record of the student with id = 3
DELETE FROM students
WHERE Id = 3;
