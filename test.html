Database Management System (DBMS)

Introduction

A Database Management System (DBMS) is software designed to store, retrieve, define, and manage data in a database. It serves as an interface between end-users and the database, ensuring data is consistently organized and remains accessible.

Objectives

  • Data integrity and security
  • Efficient data storage and retrieval
  • Minimized data redundancy
  • Concurrency control
  • Backup and recovery mechanisms

Advantages

  • Improved data sharing
  • Enhanced data security
  • Reduced data inconsistency
  • Better decision making
  • Increased productivity

Disadvantages

  • Complexity in setup and management
  • High cost of implementation
  • Performance overhead
  • Vulnerability to failures
  • Requires specialized personnel

Distributed Databases

Databases in which storage devices are not all attached to a common processor

Advantages

  • Improved performance
  • Increased reliability
  • Modular growth
  • Local autonomy

Disadvantages

  • Complexity
  • Increased costs
  • Security challenges
  • Integrity control

Normalization

Process of organizing data to minimize redundancy and dependency

Advantages

  • Reduced data redundancy
  • Improved data integrity
  • Simpler queries
  • Easier maintenance

Disadvantages

  • More complex design
  • Slower performance
  • Increased joins
  • Harder to understand
